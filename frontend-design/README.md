# Frontend Design Skill

Create distinctive, production-grade frontend interfaces with exceptional design quality.

## Overview

This skill guides the creation of visually striking, well-crafted frontend interfaces that avoid generic "AI slop" aesthetics. It combines design thinking with production-quality code implementation.

Use this skill when building:
- Web components and layouts
- Landing pages and marketing sites
- Dashboards and data visualizations
- React/Vue applications
- Interactive prototypes
- Custom web interfaces

## Sample Prompts

### Creative Direction
```text
Build a landing page for an AI startup.
Make it bold and futuristic with 3D elements and smooth animations.
Use a dark theme with neon accents (purple, cyan).
Include hero section, features grid, testimonials, and CTA.
```

### Component Design
```text
Create a beautiful input field with autocomplete suggestions.
Use a minimalist design with elegant typography.
Add smooth animations and transitions.
Include error states and loading states.
```

### Dashboard Build
```text
Build a personal finance dashboard showing spending by category.
Use a warm color palette (oranges, golds).
Show interactive charts, recent transactions, and budget progress.
Make it feel inviting and easy to understand.
```

### Redesign Existing UI
```text
This form is too boring. Redesign it with:
- Unique typography choices (not system fonts)
- A cohesive color scheme with accent colors
- Smooth interactions and transitions
- Clear visual hierarchy
- Better use of whitespace
```

### Portfolio Showcase
```text
Build a portfolio website for a photographer.
Use a grid-based layout with large, striking images.
Minimal text, maximum visual impact.
Dark background with subtle animations on scroll.
Include contact and social links.
```

## Process

1. **Understand the brief** — Purpose, audience, tone, constraints
2. **Choose an aesthetic direction** — Specific, bold, intentional
3. **Design the interface** — Layout, color, typography, motion
4. **Implement with quality code** — Production-ready, accessible, performant
5. **Refine details** — Polish, micro-interactions, edge cases

## Design Principles

- **Distinctive**: Avoid templates and generic patterns
- **Intentional**: Every design choice serves a purpose
- **Accessible**: WCAG compliance, keyboard navigation, screen readers
- **Performant**: Fast load times, smooth animations
- **Responsive**: Works beautifully on all screen sizes

## Key Guidelines

- **Typography**: Use distinctive, characterful fonts — pair display fonts with refined body text
- **Color**: Commit to a cohesive palette with dominant colors and sharp accents
- **Motion**: High-impact animations (page loads, scroll triggers) over scattered micro-interactions
- **Composition**: Embrace asymmetry, overlap, diagonal flow, and grid-breaking elements
- **Code Quality**: Production-grade implementation with proper error handling and accessibility
